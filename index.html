  • Procedural Content Generation : Dungeon Creator

    Unreal Engine 5 || Blueprints

    The assignment for this module was to create a PCG tool within UE5. I chose a dungeon creation tool to focus on due to my love of The Binding of Isaac and the assignment completed in the previous year.

    I implemented a cellular automata algorithm that would look at each tile in the grid, find its neighbouring tiles, determine what colour the tiles are and what colour they should become, and then changes the tile accordingly. The tool contains UI which allows the user to define the size of the grid and the density of the wall and floor tiles. A generate button was present to generate the grid based off of the user defined rules, and an iterate button would run the cellular automata algorithm each time the button was pressed.

  • Games Development : Procedural Dungeon Game

    C# || Unity

    This project was inspired by The Binding of Isaac. I created a game which plays very similar to TBOI, with the dungeon being procedurally generated, randomised items spawning and random enemies throughout the rooms. I created the dungeon generation system by creating arrays of room prefabs, which upon generation, would be randomly selected from the relevent array. For example, if the dungeon required a room to the left of the room, it would randomly select a room which had an open door on the right to connect them together.

    Each dungeon was generate based upon a ruleset. Depending on the floor number that the player was on, the number of rooms that could be spawned would change. The higher the floor number, the more rooms that would spawn. The dungeon would spawn the number of rooms defined, then subsequently spawning a boss room and an item room as the two furthest points from the starting room.

  • Virtual and Augmented Reality : Bowling Game

    C# || Unity

    This is the first game that I have made within VR. It is a bowling simulation type game, where the player can play a full, 10-round game of bowling. Each round allows the player to bowl twice, identical to real bowling. Subsequently, the final round of the game allows the player to bowl three times if they bowl a strike or a spare. The points system is different to real life bowling, with each pin counting as one point consistently throughout the game.

    In addition to the bowling, the environment is able to be navigated via teleportation anquors, doors can be opened and closed and objects around the environment can be picked up. This was to show off the various techniques used within VR development, such as pulleys, grabbable objects and snappable locations.
